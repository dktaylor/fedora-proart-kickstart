#### /opt/rag-stack layout
Exactly four runtime files:

`scripts/rag` → `/usr/local/bin/rag` | `systemd/rag-stack.service` → `/etc/systemd/system/`

### Key decisions
- **On-demand only**: stack starts with `rag start`, no auto-start on login
- **Data persistence**: named volumes survive `install.sh` re-runs, container removal, image updates. Only `docker volume rm` or `compose down -v` deletes data.
- **Dev vs deploy**: `~/Projects/rag-stack` = source; `/opt/rag-stack` = runtime (updated via `install.sh`)
- **Qdrant replaces ChromaDB**: existing ChromaDB KB content does not migrate — re-index with `rag index`

### Embedding model: switched to Ollama (session 2 continuation)

**Problem:** Open WebUI crashed on startup with `ValueError: No embedding model is loaded`.
- `HF_HUB_OFFLINE=1` prevented the container from resolving the model revision — `snapshot_download` needs one metadata call to find which snapshot hash corresponds to `revision=main`.
- Removing `HF_HUB_OFFLINE=1` also failed: Docker containers on this Fedora host have no outbound internet access (no masquerade to public IPs, confirmed with curl 1.1.1.1 from inside container). Firewalld docker zone showed no issue on inspection.

**Fix:** Switch embedding to Ollama — no container networking needed.
- Pulled `nomic-embed-text` via Ollama (768-dim, 274 MB)
- `docker-compose.yml`: removed HF model volume mount, added `RAG_EMBEDDING_ENGINE=ollama`, set `RAG_EMBEDDING_MODEL=nomic-embed-text`
- `models.conf`: `embedding/ollama nomic-embed-text` as active entry; HF alternatives commented
- `docs/models.md`: updated to reflect Ollama-first embedding approach
- Open WebUI started cleanly (HTTP 200 on first poll)
