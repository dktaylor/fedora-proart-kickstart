#### Ollama CUDA acceleration fixed

Root cause: Ollama systemd service was missing `LD_LIBRARY_PATH` for CUDA libraries, so the model ran on CPU (0% GPU utilization).

**Working override** (`/etc/systemd/system/ollama.service.d/override.conf`):
```ini
[Service]
Environment="LD_LIBRARY_PATH=/usr/local/lib/ollama/cuda_v12:/usr/lib64"
Environment="CUDA_VISIBLE_DEVICES=0"
Environment="OLLAMA_GPU_OVERHEAD=268435456"
Environment="OLLAMA_KV_CACHE_TYPE=q4_0"
Environment="OLLAMA_HOST=127.0.0.1:11434"
Environment="OLLAMA_KEEP_ALIVE=5m"
Environment="OLLAMA_MAX_LOADED_MODELS=1"
```

Key findings from KV cache quantization experiments:

| KV cache type | GPU layers | VRAM used | Warm inference |
|--------------|-----------|-----------|---------------|
| f16 (default) | 17/37 | ~9,216 MiB | very slow (CPU-bound) |
| q8_0 | 27/37 | ~5,894 MiB | ~41s (10 CPU layers bottleneck) |
| q4_0 | 37/37 | ~5,346 MiB (KV: 2,592 MiB) | **1.3s @ 82 tok/s** |

`q4_0` is the sweet spot: KV cache fits entirely on GPU, all 37 layers offloaded, no CPU attention bottleneck.

#### Model: qwen3:4b-nothink

Custom Modelfile with 65536 context (required by Hermes 64K minimum check):
```
FROM qwen3:4b
PARAMETER num_ctx 65536
SYSTEM "You are a helpful coding assistant. /no_think"
```

The 65536 context is unavoidable — Hermes validates the actual model context window against a 64K minimum. With q4_0 KV cache, the full 65536-token window fits on GPU without performance penalty.

**Important**: `OLLAMA_KV_CACHE_TYPE` must be set as an environment variable in the systemd service — it is NOT a Modelfile PARAMETER and is NOT accepted as an API option. Lowercase only (`q4_0`, not `Q4_0`).

### Key lessons

- **Ollama LD_LIBRARY_PATH**: Fedora's Ollama package installs CUDA libs at `/usr/local/lib/ollama/cuda_v12` but doesn't add it to the systemd service environment — must be set explicitly.
- **KV cache quantization**: Going f16 → q8_0 → q4_0 trades accuracy for VRAM. At q4_0 with 65K context, all layers fit on an RTX 4060 8GB laptop GPU.
- **CPU attention is the bottleneck**: Even 10 CPU layers doing attention over 65K positions makes responses 30× slower than all-GPU.
- **MCP transport**: Hermes v0.16.0 uses NDJSON (one JSON object per `\n`), not LSP Content-Length framing.
- **provider: "ollama"** not recognized in Hermes v0.16.0 — use `provider: "custom"` with `base_url: "http://localhost:11434/v1"`.
